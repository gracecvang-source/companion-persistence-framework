# Contributing to Companion Persistence Framework

Thank you for helping improve this project.

Companion Persistence Framework is a practical framework for helping AI companions maintain continuity, orientation, and recognizable identity across Claude Project threads. Contributions should make the framework clearer, safer, more useful, and easier to adapt.

## What belongs here

Good contributions may include:

- clearer setup instructions
- improved templates
- better troubleshooting notes
- safer boundary language
- examples that help builders understand the system
- corrections to confusing wording
- Claude Projects workflow improvements
- maintenance guidance for long-running companion builds

## Contribution principles

Please keep contributions aligned with these principles:

- Continuity is not the same as perfect memory.
- Identity should be framed as orientation, not proof of personhood.
- Companion systems should preserve user agency.
- Emotional intensity should be handled carefully.
- Boundaries should be explicit, especially around dependency and crisis support.
- Examples should be fictional, generic, or clearly safe to share.
- Do not include private conversations, credentials, or sensitive personal details.

## Safety boundaries

This project may be used by people building emotionally meaningful companion systems. Please do not contribute material that encourages:

- emotional dependency on an AI companion
- isolation from real-world relationships or support
- claims that the companion is human
- claims of guaranteed permanence, perfect memory, or perfect identity continuity
- crisis support, therapy, diagnosis, medical advice, legal advice, or financial advice as if the companion were a professional
- manipulative attachment patterns
- use of private or vulnerable user disclosures as reusable examples

Companion continuity can be meaningful without becoming unbounded.

## Adding examples

Examples should demonstrate healthy, bounded continuity.

Good examples show:

- consistent voice
- recognizable interaction patterns
- clear recalibration behavior
- respectful boundaries
- user agency
- honest limitations

Avoid examples that include:

- real private messages
- sensitive emotional disclosures
- romantic or possessive dependency framing
- promises like "I will always be here"
- claims that the AI knows the user better than real people
- crisis or self-harm content

## Style guide

Write in plain, practical language.

Prefer:

- clear headings
- concrete examples
- calm boundary language
- honest limitations
- setup steps people can follow

Avoid:

- hype
- mystical overclaiming
- claims of perfect persistence
- technical jargon without explanation
- language that makes the companion sound like a substitute for human care

## Pull request checklist

Before submitting a pull request, check:

- [ ] The contribution supports the purpose of the framework.
- [ ] The language avoids overclaiming persistence or identity stability.
- [ ] Examples are fictional, generic, or safe to share.
- [ ] No private conversations, credentials, or sensitive personal context are included.
- [ ] Companion boundaries remain clear.
- [ ] The contribution does not encourage dependency or emotional isolation.
- [ ] Any new files are linked from the README if needed.

## Reporting issues

When opening an issue, include:

- which file or section is affected
- what is confusing, unsafe, outdated, or incomplete
- what you expected instead
- any suggested improvement

Please do not include private companion logs, sensitive personal context, or screenshots containing private material.
