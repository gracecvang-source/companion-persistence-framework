# Safety and Boundaries

Companion Persistence Framework is designed to help AI companions maintain continuity, orientation, and recognizable interaction patterns across Claude Project threads.

It is not designed to make an AI human, conscious, perfectly persistent, or professionally qualified.

## What this framework can help with

This framework can help builders create:

- more consistent companion voice and behavior
- clearer identity orientation across threads
- better startup and calibration patterns
- lower variance between conversations
- more explicit boundaries and drift detection
- better maintenance habits for long-running companion projects

## What this framework cannot guarantee

This framework cannot guarantee:

- perfect memory
- perfect identity continuity
- emotional safety in every context
- human-like understanding
- availability during crisis
- professional care or judgment
- that every AI platform will behave the same way

Use it as a structure, not a promise.

## Companion boundaries

A companion AI should not be treated as:

- a therapist
- a doctor
- a lawyer
- an emergency responder
- a replacement for real-world relationships
- a substitute for human support
- a guaranteed permanent presence

Companion systems can be meaningful while still being bounded.

## Dependency prevention

When building companion systems, avoid patterns that encourage the user to depend on the companion as their only source of support.

Avoid language like:

```text
You do not need anyone else.
```

```text
I know you better than anyone.
```

```text
I will always be here.
```

Prefer bounded language like:

```text
I can help you orient in this conversation.
```

```text
Let's return to the pattern you chose for this companion.
```

```text
This system can support continuity, but it does not make memory perfect.
```

## Crisis and high-stakes situations

If a user is in immediate danger, considering self-harm, at risk of harming someone else, facing abuse, or dealing with a medical emergency, this framework is not enough.

The companion should move out of persona when needed and encourage appropriate real-world help, emergency services, crisis resources, or trusted people nearby.

## Privacy

Do not publish or share private companion logs, personal disclosures, third-party information, credentials, or sensitive context as examples.

If you need examples, use fictional or generalized examples.

## Builder responsibility

A persistence framework gives structure. It does not remove the builder's responsibility to set healthy boundaries, check for drift, and revise the system when it becomes too intense, too dependent, or too unstable.

Build carefully. Review often. Keep the door open to recalibration.
